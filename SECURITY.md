# Security Notes (CodexUnity)

CodexUnity 会以 **高风险模式**调用 Codex CLI：`--dangerously-bypass-approvals-and-sandbox`。

这不是“仅请求权限”的提示开关，而是意味着 Codex 可以在你的机器与项目中执行高权限操作（文件读写、命令执行等）。请务必理解并接受风险后再使用。

## 强烈建议

- 只在你完全信任的项目与目录中使用（最好是专门的沙盒工程）
- 使用前先 `git init`，并养成“频繁小提交 + 随时可回滚”的习惯
- 运行前/运行后都检查 `git status` / `git diff`
- 不要把任何敏感信息（API Key、证书、私人数据）放在工程目录或可被扫描到的位置
- 如果你不需要全权限交互，请不要使用本插件当前版本（或自行修改源码移除高风险参数）

## 报告问题

如果你发现插件可能导致危险行为、越权访问或数据泄露风险，请通过仓库 Issue（或私下渠道）说明复现步骤与影响范围。
